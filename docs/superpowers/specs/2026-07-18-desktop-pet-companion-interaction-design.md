# 桌宠陪伴互动与 AI 对话设计

## 目标

将桌宠从任务状态展示工具升级为具有持续状态、动作反馈和可选 AI 对话能力的陪伴型桌宠，同时修复 Windows 拖动时窗口追赶历史坐标的问题。

## 范围

本阶段包含：

- 点击、双击、拖动和抚摸四种直接互动。
- 喂食、睡觉和唤醒操作。
- 心情、饱食度和精力三项持续数值。
- 正常、开心、委屈、饥饿、困倦和睡眠六种基础形态。
- 任务成功、任务失败和额度不足等短暂业务动作。
- OpenAI 兼容模型配置与主动对话。
- Windows 拖动坐标链路修复。

本阶段不包含物品商城、成长等级、账号间宠物状态同步、语音输入输出和云端对话历史。

## 总体架构

### InteractionEngine

Renderer 中的 `InteractionEngine` 负责将指针输入转换为标准互动事件：

- 单击：短按且移动距离小于 5 像素。
- 双击：两次单击间隔不超过 300 毫秒。
- 拖动：按下后的前 350 毫秒内移动距离达到 5 像素时开始窗口拖动。
- 抚摸：静止按住宠物达到 350 毫秒后锁定抚摸候选，在宠物区域内持续往返移动，累计轨迹达到 80 像素且主轴方向反转至少 2 次时确认抚摸。

互动事件只描述用户意图，不直接修改宠物数值或操作窗口。

### PetLifeStore

`PetLifeStore` 管理可持久化的宠物生命状态：

- `mood`：心情值，范围 0 至 100。
- `satiety`：饱食度，范围 0 至 100。
- `energy`：精力值，范围 0 至 100。
- `sleeping`：是否处于睡眠状态。
- `lastCalculatedAt`：上次结算时间。

应用启动和定时更新时根据时间差结算衰减。系统时间回拨时按零时间差处理，超长离线结算设置最大跨度，避免一次性产生极端状态。

首版数值规则如下：

| 项目 | 规则 |
| --- | --- |
| 饱食度 | 清醒时每小时减少 2，单次喂食增加 25 |
| 精力 | 清醒时每小时减少 1.5，睡眠时每小时增加 8 |
| 心情 | 连续 6 小时无互动后每小时减少 1 |
| 点击 | 心情增加 1，每 10 分钟最多计入 3 次 |
| 双击 | 心情增加 3，每 10 分钟最多计入 2 次 |
| 抚摸 | 每 2 秒心情增加 1，单次最多增加 5 |
| 任务成功 | 心情增加 2 |
| 任务失败 | 心情减少 2 |

所有数值在写入时限制到 0 至 100。离线结算最大跨度为 72 小时。

### PetActionController

`PetActionController` 接收互动、业务事件和生命状态，选择当前动作。优先级从高到低为：

1. 拖动与抚摸动作。
2. 喂食、入睡和唤醒动作。
3. 任务成功与任务失败动作。
4. 额度不足提示。
5. 基础形态动作。
6. 空闲随机动作。

临时动作包含明确持续时间和完成回调。动作完成后重新计算基础形态，避免回到过期状态。

### DialogueCoordinator

`DialogueCoordinator` 位于 Electron 主进程，负责主动对话排期、事件去重、冷却和模型调用：

- 空闲主动对话每 15 至 30 分钟随机排期一次。
- 任务开始、任务成功、任务失败和额度明显变化时触发事件对话。
- 同一业务事件通过稳定事件键只触发一次。
- 静默时段暂停主动对话，事件只更新宠物动作。
- 模型请求失败后进入 5 分钟冷却，并使用本地预设台词。

额度事件在跨越 20%、跨越 5%、耗尽或从耗尽状态恢复时触发。相同阈值和方向组合在同一登录 generation 中只触发一次。

### LLMService 与配置存储

`LLMService` 由 Electron 主进程调用 OpenAI 兼容接口。配置包含：

- Base URL。
- API Key。
- Model。
- 主动对话开关。
- 静默时段。

API Key 使用 Electron `safeStorage` 加密保存。Renderer 只能提交新配置、清除配置以及读取 `configured` 和脱敏摘要。日志中不记录 API Key、Authorization header、完整请求正文和模型原始错误响应。

## 行为设计

### 直接互动

- 点击：宠物看向用户并挥手，显示一条短气泡。
- 双击：宠物开心跳跃并增加少量心情值。
- 抚摸：宠物闭眼享受，按有效持续时间增加心情值；单次收益设置上限。
- 拖动：宠物切换为抓起姿态，窗口跟随当前光标；松开后播放轻微回弹。

### 生活操作

- 喂食：播放取食、咀嚼和满足动作，提高饱食度并小幅提高心情值。
- 睡觉：先播放困倦过渡，再进入睡眠形态；睡眠期间精力逐步恢复。
- 唤醒：点击睡眠中的宠物触发唤醒动作。精力过低时允许宠物短暂赖床并显示本地台词。

### 基础形态选择

基础形态按以下顺序确定：

1. `sleeping` 为真时选择睡眠。
2. 饱食度进入低阈值时选择饥饿。
3. 精力进入低阈值时选择困倦。
4. 心情进入低阈值时选择委屈。
5. 心情进入高阈值时选择开心。
6. 其余情况选择正常。

饥饿在饱食度不高于 25 时进入，高于 35 时退出；困倦在精力不高于 20 时进入，高于 35 时退出；委屈在心情不高于 30 时进入，高于 40 时退出；开心在心情不低于 70 时进入，低于 60 时退出。滞回区间避免数值在边界附近导致形态频繁切换。

### 状态视觉

六种基础形态使用同一猴子造型体系，通过眼睛、嘴型、姿势、道具和动作节奏区分。任务和额度动作保持现有业务语义，并作为临时覆盖层播放。所有持续动画支持 `prefers-reduced-motion`，降级后保留可辨识的静态表情。

## 拖动修复

Renderer 继续负责 5 像素手势阈值和指针捕获，只向主进程发送拖动开始、移动通知和结束事件。主进程通过 `screen.getCursorScreenPoint()` 读取 Electron DIP 坐标：

1. 开始拖动时记录当前 DIP 光标和窗口边界。
2. 移动通知到达时读取最新 DIP 光标，计算相对起点的窗口位置。
3. 合并高频移动通知，只应用最新位置。
4. 结束拖动时应用最后一次当前光标位置并关闭会话。
5. 继续使用现有多显示器工作区覆盖与边界限制算法。

主进程不再使用 Renderer 传入的 `screenX` 和 `screenY` 计算窗口位置，从而消除历史坐标追赶和坐标单位差异。

## AI 对话

### 上下文

发送给模型的上下文限制为：

- 当前基础形态和三项生命数值的分段描述。
- 最近一次用户互动。
- 最近期任务事件。
- 当前额度百分比区间。
- 最近 10 条宠物对话。

上下文不包含 session Cookie、用户密码、验证码 token、任务原始输入和 API Key。

### 输出约束

- 模型输出按纯文本渲染。
- 单条气泡限制长度，超出后安全截断。
- 不解析 HTML、Markdown 链接和可执行内容。
- 气泡展示完成后自动消失，用户点击可提前关闭。
- 用户可清空本地对话历史并关闭主动对话。

### Base URL 校验

生产环境只接受 HTTPS URL。开发环境允许 `localhost`、`127.0.0.1` 和 `[::1]`。URL 禁止携带用户名、密码和 fragment。请求只由主进程发起。

## 持久化与恢复

宠物生命状态和非敏感模型设置存入 Electron store。API Key 单独使用 `safeStorage` 加密。启动时先恢复快照，再按 `lastCalculatedAt` 结算离线变化。损坏或越界数据使用安全默认值恢复，并保留可诊断的非敏感错误信息。

## 错误处理

- 拖动会话异常时终止当前会话，窗口停留在最后有效位置。
- 生命周期持久化失败时继续使用内存状态，并向用户显示一次非阻塞提示。
- 模型配置无效时在保存前返回字段级错误。
- 模型请求超时、限流或网络失败时使用本地台词并进入冷却。
- 动作资源加载失败时回退到对应基础形态的静态资源。

## 测试与验收

### 自动化测试

- 手势：单击、双击、拖动、抚摸之间互斥且阈值正确。
- 生命周期：衰减、睡眠恢复、离线结算、边界值和系统时间回拨。
- 形态：六种基础形态、滞回区间和临时动作恢复。
- 动作：优先级、持续时间、覆盖和完成后重算。
- 对话：15 至 30 分钟排期、静默时段、事件去重、失败冷却和本地回退。
- 安全：配置脱敏、Key 加密、Base URL 校验和 Renderer 隔离。
- 拖动：延迟移动、快速移动、结束位置、多显示器边界和过期会话。

### Windows 实机验收

- 100% 显示缩放下快速拖动时窗口紧跟光标且停止后不继续移动。
- 125% 和 150% 显示缩放下拖动距离一致。
- 多显示器和负坐标显示器之间移动稳定。
- 点击、双击、拖动和抚摸能够稳定区分。
- 喂食、睡觉、唤醒及六种基础形态视觉明确。
- 配置有效模型后，主动对话和任务/额度事件对话按规则触发。
- 清除模型配置后，宠物继续使用本地互动和预设台词。

## 发布策略

实现拆为两个连续里程碑：

1. 互动核心：拖动修复、手势、生命数值、六种基础形态、喂食和睡眠。
2. AI 对话：安全配置、模型调用、主动排期、事件对话和气泡历史。

每个里程碑独立完成自动化验证和 Windows 安装包验收，便于隔离交互回归与外部模型服务问题。
