# 桌宠交互与状态表现设计

## 目标

完善 Windows 桌宠的核心交互和状态反馈：猴子可直接拖动，透明区域允许操作下方窗口，环绕状态卡清晰展示任务、额度和签到信息，五种猴子表情与动作由真实业务状态驱动。

本设计是第一阶段。自定义 AI 模型配置、加密凭据和动作决策作为第二阶段独立设计与实施。

## 窗口交互

### 直接拖动

- 用户在猴子主体上按住左键并移动时拖动窗口。
- 指针位移达到 5 像素后判定为拖动，释放时不触发展开或收起。
- 位移低于阈值时判定为单击，切换环绕状态卡。
- 主进程负责移动窗口，并将窗口边界限制在当前显示器工作区内。

### 点击穿透

- 猴子、状态卡和按钮属于交互区域，窗口接收鼠标事件。
- 透明空白区域开启 Electron 鼠标事件穿透，使用户可以点击下方应用。
- Renderer 只上报命中状态和拖动坐标；主进程验证 IPC 来源、参数范围和窗口状态。

### 窗口尺寸

- 折叠态使用紧凑窗口，仅容纳猴子和动作效果。
- 展开态扩大窗口，围绕猴子展示状态卡。
- 切换尺寸时保持猴子的屏幕锚点稳定，并将新窗口边界限制在显示器工作区。

## 环绕状态卡

采用已选定的 C 方案，以猴子为视觉中心展示三类浮动卡：

- 任务卡：活跃任务数量、标题和当前状态；任务结束后显示最近一次完成或失败事件。
- 额度卡：今日剩余额度和低额度提示。
- 签到卡：可签到、提交中、签到成功、今日已签到或失败状态。

展开态提供手动刷新、打开 MonkeyCode 和退出登录入口。按钮及卡片具备清晰的 hover、pressed、disabled 和 focus 状态。

## 任务自动监控

### 活跃任务轮询

- 每 15 秒查询 `pending,processing` 活跃任务。
- 后端并发任务上限为 3，桌宠只维护当前活跃任务集合。
- 首次查询建立任务状态基线，不播放历史任务事件。

### 终态确认

- 当已知活跃任务从活跃列表消失时，按任务 ID 查询任务详情。
- `finished` 触发成功事件，`error` 触发失败事件。
- 成功或失败事件携带任务 ID、标题、终态和发生时间。
- 同一任务终态只触发一次；登录切换、退出和认证失效时清空检测基线。

### 瞬时动作

- 成功或失败动作持续 8 秒。
- 8 秒结束后，根据当前活跃任务、额度和网络状态恢复基础状态。
- 新终态事件到达时替换当前瞬时事件并重新计时。

## 数据刷新策略

### 签到

- 登录成功后查询一次当天签到状态。
- 以账号标识和本地日期缓存查询结果。
- 已签到时直接显示“今日已签到”并禁用重复签到。
- 签到成功后立即更新缓存和钱包。
- 本地日期变化后使缓存失效并重新查询。

### 钱包

- 登录成功、签到成功、任务进入终态时刷新钱包。
- 每 5 分钟执行一次兜底刷新。
- 手动刷新同时更新活跃任务、签到和钱包。

## 五种猴子状态

- `IDLE`：平静表情和呼吸、眨眼动作。
- `WORKING`：专注表情和持续工作动作。
- `SUCCESS`：开心表情、跳跃和短暂庆祝粒子，持续 8 秒。
- `ERROR`：沮丧表情和轻微摇晃动作，持续 8 秒。
- `QUOTA_LOW`：疲惫表情和低频下沉动作。

每种状态使用独立 SVG 资源和对应 CSS 动画。系统减少动画偏好开启时，动作降级为静态表情。

## 状态优先级

瞬时任务事件优先于基础状态：

1. 最近 8 秒的任务失败或完成事件。
2. 存在 `pending` 或 `processing` 任务时为 `WORKING`。
3. 今日剩余额度低于 10% 时为 `QUOTA_LOW`。
4. 其余情况为 `IDLE`。

离线状态保留最近业务数据，并在环绕卡中显示离线提示。离线本身不覆盖最近一次可解释的猴子业务状态。

## 错误处理

- 任务详情确认失败时保留任务待确认状态，在下一轮重试，避免误报完成或失败。
- 签到状态查询失败时显示“签到状态暂不可用”，允许手动刷新。
- 拖动或穿透 IPC 参数异常时拒绝操作，保持当前窗口位置和命中状态。
- 所有错误信息避免包含会话、密码、验证码 token 或用户配置凭据。

## 测试与验收

### 自动测试

- 任务首次基线不触发事件。
- `pending/processing` 转换为 `finished/error` 时只触发一次终态事件。
- 登录代际变化后旧请求和旧任务事件失效。
- 瞬时状态 8 秒后恢复正确基础状态。
- 签到状态按账号和日期缓存，跨天重新查询。
- 已签到状态禁用按钮并显示明确文案。
- 5 像素拖动阈值正确区分单击和拖动。
- 窗口移动和展开尺寸被限制在显示器工作区。
- 五种状态均映射到独立资源，减少动画偏好可关闭动作。

### Windows 实机验收

- 按住猴子可拖动，短按可展开或收起。
- 透明区域可以点击下方网页或应用。
- 环绕卡所有按钮可点击，窗口不会遮挡卡片外区域。
- 任务开始、完成和失败在一个轮询周期内被检测并展示正确动作。
- 已签到账号启动后显示“今日已签到”。
- 五种状态的表情与动作清晰可区分。

## 第二阶段边界

自定义 AI 仅能选择预定义动作和安全文案，不能直接执行系统命令、构造任意 IPC 或读取会话凭据。模型 API Key 由用户在应用内配置，使用 Electron `safeStorage` 加密保存，且不进入代码、日志或构建产物。第二阶段将单独定义调用时机、预算、隐私数据范围、超时降级和模型配置界面。
