# 桌宠状态动作验收展厅设计

## 背景

桌宠 1.2.0 已在生产界面中使用 15 张 SVG 资源，并通过贴图复用和 CSS 动画组合出 17 个运行时动作。Windows 实机验收需要一个可通过浏览器访问的总览页面，用于集中检查每个状态的形象、动画、资源映射和降级行为。

## 目标

- 通过独立 Web 入口同时展示全部 17 个运行时动作。
- 复用生产 `MonkeySprite`、动作映射和动画 class，保证展厅效果与 EXE 同源。
- 明确展示全部 15 张打包 SVG，包括当前动作映射未引用的 `idle.svg` 和 `working.svg`。
- 提供响应式布局，支持桌面端和移动端浏览验收。
- 保持 Electron 正常启动、认证和桌宠交互流程不变。

## 入口与隔离

现有 Vite 应用增加查询参数入口 `/?gallery=1`。应用启动时优先检查该参数：

- 参数为 `gallery=1` 时直接渲染验收展厅，不访问 `window.electronAPI`。
- 其他情况继续执行现有 `App`，保持 EXE 行为不变。

展厅属于验收工具，不进入桌宠窗口导航，也不增加 Electron IPC。生产构建可包含该页面代码，只有显式 URL 参数能够进入。

## 页面结构

页面采用深色展厅视觉，内容分为三部分：

1. 顶部概览：显示版本 `1.2.0`、17 个运行时动作、15 张打包贴图和用途说明。
2. 动作网格：按生命形态、互动动作、生活动作、业务状态分组，同时播放全部动作。
3. 资源清单：展示 15 张 SVG 文件及引用情况，明确标记复用资源与辅助资源。

动作网格包含以下分组：

- 生命形态：`normal`、`happy`、`sad`、`hungry`、`sleepy`、`sleeping`。
- 互动动作：`waving`、`celebrating`、`petting`、`dragging`、`dropping`。
- 生活动作：`eating`、`falling-asleep`、`waking`。
- 业务状态：`task-success`、`task-error`、`quota-low`。

每张动作卡片展示生产 `MonkeySprite`、中文名称、内部 action key、实际 SVG 文件名和资源复用标记。动画持续播放，有限次数的生产动画在卡片内周期性重新触发，便于连续验收。

## 组件与数据

新增独立 `PetActionGallery` 组件。展厅元数据使用带类型约束的分组数组维护，action key 必须满足 `PetAction` 类型。

贴图 URL、动画 class 和无障碍文案继续读取 `MonkeySprite.tsx` 导出的生产映射。展厅不复制动作与资源映射。资源清单由显式的 15 项文件清单驱动，并根据动作映射计算每张资源的引用动作。

页面样式独立使用 gallery 前缀，避免影响桌宠窗口现有尺寸、透明背景、鼠标穿透和动画规则。动作图像区域继续使用生产 `.pet-sprite` 与 `.pet-*` class。

## 动画重播

`waving`、`celebrating`、`petting`、`dropping`、`waking`、`task-success` 和 `task-error` 使用有限动画次数。展厅每隔固定周期更新卡片实例 key，使这些动画重新开始；无限循环动作保持自然连续播放。

重播只存在于展厅组件，不修改生产动作时长和 `PetShell` 的动作调度。

## 错误处理

- 动作贴图加载失败时继续使用 `MonkeySprite` 的生产 fallback，回退到 `normal`。
- 资源清单中的图片加载失败时显示文件名与“资源加载失败”状态，保留卡片布局。
- 展厅不依赖 Electron API，可直接在普通浏览器和 Vite 预览环境运行。
- `prefers-reduced-motion` 生效时停止展厅装饰动画，并沿用生产样式中的降动效规则。

## 测试

- 验证查询参数路由只在 `gallery=1` 时进入展厅。
- 验证四个分组完整且只包含 17 个 `PetAction`。
- 验证每个动作卡片使用生产映射对应的 SVG 和动画 class。
- 验证 15 张打包资源全部出现在资源清单中。
- 验证 `idle.svg` 和 `working.svg` 标记为打包辅助资源。
- 验证普通应用入口继续渲染原有 `App`。
- 运行 `npm run verify`，随后启动 Vite 服务并检查桌面端与移动端布局。

## 验收标准

- 一个 Web 地址可同时观察全部 17 个运行时动作。
- 页面完整列出 15 张打包 SVG，资源复用关系清晰可见。
- 展厅动画与 EXE 使用同一组件、映射和 CSS。
- 展厅在普通浏览器中不触发 Electron API 错误。
- 普通 EXE 启动和桌宠交互行为保持不变。
- 类型检查、测试、Electron bundle 验证和 Vite 构建全部通过。

## 范围外

- 修改桌宠贴图、美术风格或生产动画参数。
- 将展厅加入 EXE 的用户界面或菜单。
- 提供在线编辑、上传或替换贴图功能。
- 改变动作优先级、手势识别和生命状态规则。
